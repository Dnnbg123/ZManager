# 🚧 Статус разработки ZManager (Vue 3 Frontend)

Дата обновления: 2024-05-24
Цель: Создание высокопроизводительного файлового менеджера с архитектурой уровня IDE (Multi-Split View).

---

## ✅ Выполнено (MVP Архитектура)

### 1. Ядро системы разделения экрана (Multi-Split Engine)
Реализована рекурсивная древовидная структура layout, позволяющая создавать неограниченное количество панелей.
- **State Management (`splitView.store.ts`)**:
  - Рекурсивная структура данных (`LayoutNode` = `PanelNode` | `SplitNode`).
  - Логика добавления/удаления панелей (`splitPanel`, `closePanel`).
  - Поддержка горизонтального и вертикального разделения.
  - Отслеживание активной панели.
- **Рендеринг (`SplitViewManager.vue`)**:
  - Рекурсивный компонент для отрисовки дерева layout.
  - Динамическое переключение между контейнером и панелью.

### 2. Компоненты UI Панели
- **`SplitContainer.vue`**:
  - Контейнер для группировки панелей.
  - **Drag-to-resize**: Базовая реализация изменения размеров панелей перетаскиванием разделителя.
  - Поддержка обоих направлений (row/col).
- **`FilePanel.vue`**:
  - Обертка для отдельной рабочей области.
  - Подсветка активной панели (focus ring).
  - Интеграция всех подкомпонентов (Tabs, Toolbar, Breadcrumb, List).
- **`PanelTabs.vue`**:
  - Интерфейс вкладок внутри панели.
  - Переключение, закрытие, добавление новых вкладок.
  - Визуальная индикация активного таба.
- **`PanelToolbar.vue`**:
  - Кнопки управления: Split Horizontal, Split Vertical, Close Panel.
  - Минималистичный дизайн.
- **`Breadcrumb.vue`**:
  - Отображение текущего пути навигации.
- **`FileList.vue`**:
  - Заглушка списка файлов с моковыми данными.
  - Подготовка места для внедрения виртуализации (`vue-virtual-scroller`).

### 3. Интеграция и Стилизация
- **`App.vue`**: Полная замена старого контента на новый `SplitViewManager`.
- **Tailwind CSS**: Настроена темная тема (Neutral palette), стили скроллбаров, утилиты для flex/grid layout.
- **TypeScript**: Строгая типизация всех компонентов и store (без `any`).

## 🛠 Технические долги и риски

- **Resize Logic**: Текущая реализация изменения размеров работает корректно только для простых случаев (2 панели). Для сложных вложенных структур требуется алгоритм пересчета весов соседей.
- **Performance**: При глубокой вложенности сплитов (>10 уровней) могут потребоваться оптимизации рендеринга Vue (shallowRef, markRaw).
- **Tauri v2 API**: Необходимо строго следовать актуальной документации Tauri v2 для IPC вызовов, так как API мог измениться по сравнению с v1.
